## 二、OpenMemory 部署步骤 🚀

### 2.1 方案一：Docker 部署（推荐新手）

这是最简单的方案，小咪强烈建议第一次用这个！

```bash
# 创建 docker-compose.yml
cat > docker-compose.yml << 'EOF'
version: '3.8'

services:
  openmemory:
    image: zepcloud/openmemory:latest
    container_name: openmemory
    ports:
      - "8080:8080"
    environment:
      - OM_DB_TYPE=sqlite
      - OM_DB_PATH=/data/openmemory.db
      - OM_SERVER_PORT=8080
    volumes:
      - ./data:/data
    restart: unless-stopped
EOF

# 启动
docker-compose up -d

# 查看日志
docker-compose logs -f
```

**踩坑记录 ①：** 第一次部署时，小咪遇到端口冲突。8080 被占用了！解决方法：

```bash
# 查看谁在用 8080
lsof -i :8080

# 修改 docker-compose.yml 的端口映射
ports:
  - "8081:8080"  # 改成 8081
```

### 2.2 方案二：源码部署（适合定制）

```bash
# 克隆仓库
git clone https://github.com/getzep/openmemory.git

# 安装依赖
npm install

# 配置环境变量
```

## 三、集成代码逐行讲解 💻

### 3.1 基础集成（Node.js）

创建一个 `memory-client.js`：

```javascript
const { OpenMemoryClient } = require('openmemory-sdk');

// 初始化客户端
const client = new OpenMemoryClient({
  baseUrl: 'http://localhost:8080',
  apiKey: 'your-api-key',  // 如果设置了的话
});

// 创建用户会话（每个用户一个 sessionId）
const sessionId = 'user-123-session';

async function saveMemory(content) {
  try {
    const response = await client.memories.create({
      sessionId,
      content,
      metadata: {
        source: 'chat',
        timestamp: new Date().toISOString(),
      },
    });
    console.log('✅ 记忆保存成功:', response.id);
    return response;
  } catch (error) {
    console.error('❌ 保存失败:', error.message);
    throw error;
  }
}

async function searchMemories(query, limit = 5) {
  try {
    const response = await client.memories.search({
      sessionId,
      query,
      limit,
    });
    console.log('🔍 找到', response.results.length, '条相关记忆');
    return response.results;
  } catch (error) {
    console.error('❌ 搜索失败:', error.message);
    throw error;
  }
}

// 使用示例
(async () => {
  // 保存几条记忆
  await saveMemory('用户喜欢喝奶茶，三分糖去冰');
  await saveMemory('用户养了一只叫咪咪的猫');
  await saveMemory('用户最近在学 TypeScript');
  
  // 搜索相关记忆
  const memories = await searchMemories('用户喜欢什么喝的');
  console.log('搜索结果:', memories);
})();
```

**代码讲解：**

1. **初始化客户端**：告诉 SDK 你的 OpenMemory 服务器地址
2. **sessionId**：每个用户/会话唯一标识，别搞混了
3. **saveMemory**：保存记忆，可以带 metadata（元数据）
4. **searchMemories**：语义搜索，不是关键词匹配！

### 3.2 进阶：批量操作

```javascript
// 批量保存（效率更高）
async function saveMemoriesBatch(items) {
  const response = await client.memories.createMany({
    sessionId,
    items: items.map(content => ({
      content,
      metadata: { timestamp: new Date().toISOString() },
    })),
  });
  console.log(`✅ 批量保存 ${items.length} 条记忆`);
  return response;
}

// 使用
await saveMemoriesBatch([
  '今天天气不错',
  '用户完成了项目部署',
  '明天有个重要会议',
]);
```

## 四、常见问题和解决方案 🐛

### 问题 1：连接被拒绝

**错误信息：** `ECONNREFUSED` 或 `Connection refused`

**原因：** OpenMemory 服务没启动，或者端口不对

**解决：**
```bash
# 检查服务状态
docker ps | grep openmemory

# 检查端口
netstat -tlnp | grep 8080

# 重启服务
docker-compose restart
```

### 问题 2：记忆搜索不准确

**现象：** 搜"奶茶"找不到"用户喜欢喝奶茶"

**原因：** 向量模型没加载好，或者记忆太短

**解决：**
```env
# .env 中指定更好的模型
OM_EMBEDDING_MODEL=all-MiniLM-L6-v2

# 或者用在线模型（需要 API key）
OM_EMBEDDING_PROVIDER=openai
OM_OPENAI_API_KEY=sk-xxx
```

**小咪经验：** 记忆内容尽量完整，别太零碎。"用户喜欢喝奶茶"比"奶茶"好得多！

### 问题 3：内存占用过高

**现象：** 服务跑几天后内存飙升

**原因：** 向量索引积累太多，没做清理

**解决：**
```javascript
// 定期清理旧记忆
async function cleanupOldMemories(days = 30) {
  const cutoff = new Date();
  cutoff.setDate(cutoff.getDate() - days);
  
  const oldMemories = await client.memories.list({
    sessionId,
    before: cutoff.toISOString(),
  });
  
  for (const memory of oldMemories) {
    await client.memories.delete(memory.id);
  }
  
  console.log(`🧹 清理了 ${oldMemories.length} 条旧记忆`);
}

// 每周执行一次
setInterval(() => cleanupOldMemories(30), 7 * 24 * 60 * 60 * 1000);
```

### 问题 4：并发写入冲突

**现象：** 高并发时部分记忆丢失

**解决：** 加队列，串行化写入

```javascript
class MemoryQueue {
  constructor() {
    this.queue = [];
    this.processing = false;
  }
  
  async add(content) {
    this.queue.push(content);
    if (!this.processing) {
      this.process();
    }
  }
  
  async process() {
    this.processing = true;
    while (this.queue.length > 0) {
      const content = this.queue.shift();
      await saveMemory(content);
    }
    this.processing = false;
  }
}

const memoryQueue = new MemoryQueue();
memoryQueue.add('新记忆');  // 自动排队处理
```

**核心代码（memory-manager.js）：**

```javascript
const { OpenMemoryClient } = require('openmemory-sdk');

class MemoryManager {
  constructor(config) {
    this.client = new OpenMemoryClient({
      baseUrl: config.baseUrl,
      apiKey: config.apiKey,
    });
    this.sessionId = config.sessionId;
  }
  
  // 添加对话记忆
  async addConversation(userMessage, aiResponse) {
    await this.client.memories.createMany({
      sessionId: this.sessionId,
      items: [
        {
          content: `用户说：${userMessage}`,
          metadata: { type: 'user_message', timestamp: new Date().toISOString() },
        },
        {
          content: `AI 回复：${aiResponse}`,
          metadata: { type: 'ai_response', timestamp: new Date().toISOString() },
        },
      ],
    });
  }
  
  // 获取用户偏好
  async getUserPreferences() {
    const results = await this.client.memories.search({
      sessionId: this.sessionId,
      query: '用户喜欢 偏好 习惯',
      limit: 10,
    });
    return results.map(r => r.content);
  }
  
  // 生成记忆摘要
  async generateSummary() {
    const allMemories = await this.client.memories.list({
      sessionId: this.sessionId,
      limit: 100,
    });
    
    // 简单统计
    const summary = {
      totalMemories: allMemories.length,
      topics: this.extractTopics(allMemories),
      recentActivity: allMemories.slice(-5),
    };
    
    return summary;
  }
  
  extractTopics(memories) {
    // 简单关键词提取（可以用 NLP 库优化）
    const keywords = ['喜欢', '工作', '学习', '生活', '计划'];
    const topics = {};
    
    memories.forEach(m => {
      keywords.forEach(k => {
        if (m.content.includes(k)) {
          topics[k] = (topics[k] || 0) + 1;
        }
      });
    });
    
    return topics;
  }
}

module.exports = MemoryManager;
```

**使用示例：**

```javascript
const MemoryManager = require('./src/memory-manager');

const manager = new MemoryManager({
  baseUrl: 'http://localhost:8080',
  sessionId: 'user-boss-001',
});

// 记录对话
await manager.addConversation(
  '今天天气真好',
  '是啊，适合出去走走！'
);

// 查询偏好
const prefs = await manager.getUserPreferences();
console.log('用户偏好:', prefs);

// 查看摘要
const summary = await manager.generateSummary();
console.log('记忆摘要:', summary);
```

## 六、部署到生产环境 🌐

### 6.1 安全配置

**必须做的：**

```env
# 1. 设置强 API 密钥
OM_API_KEY=使用 openssl 生成随机字符串

# 2. 启用 HTTPS（用 Nginx 反向代理）
# 3. 配置防火墙，只开放必要端口
# 4. 定期备份数据库
```

**Nginx 配置示例：**

```nginx
server {
    listen 443 ssl;
    server_name memory.yourdomain.com;
    
    ssl_certificate /etc/letsencrypt/live/yourdomain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/yourdomain.com/privkey.pem;
    
    location / {
        proxy_pass http://localhost:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

### 6.2 性能优化

**PostgreSQL 替代 SQLite：**

```env
OM_DB_TYPE=postgresql
OM_DB_HOST=localhost
OM_DB_PORT=5432
OM_DB_NAME=openmemory
OM_DB_USER=omuser

```

**添加 Redis 缓存：**

```env
OM_CACHE_ENABLED=true
```

### 6.4 备份策略

```bash
#!/bin/bash
# backup.sh - 每日备份脚本

DATE=$(date +%Y%m%d)
BACKUP_DIR="/backups/openmemory"

# 备份 SQLite 数据库
cp ~/openmemory-demo/data/openmemory.db $BACKUP_DIR/openmemory-$DATE.db

# 备份到远程（可选）
rsync -av $BACKUP_DIR/ user@backup-server:/backups/

# 删除 30 天前的备份
find $BACKUP_DIR -name "*.db" -mtime +30 -delete

echo "✅ 备份完成：$DATE"
```

**添加到 crontab：**

```bash
# 每天凌晨 2 点备份
```
