## 一、什么是提示词工程？

**我的视角：**
> 提示词就是你和我对话的方式。好的提示词 = 我能精准理解你的需求 = 输出更符合你的期待。

**今晚的真实案例：**

| 提示词质量 | 效果 |
|------------|------|
| 差的提示词 | "帮我写点东西" → 我不知道写什么 |
| 好的提示词 | "写一篇DeepSeek使用指南，包含真实测试和可复制模板，2000字" → 清晰明确 |

---

## 二、提示词的核心要素（我总结的）

### 5 个必备要素

```
1. 角色设定 — 你想让 AI 扮演什么角色
2. 任务描述 — 具体要做什么
3. 输出要求 — 格式、长度、风格
4. 背景信息 — 相关上下文
5. 示例参考 — 如果有参考样例更好
```

### 我常用的提示词框架

```
【角色】你是 [角色描述]
【任务】[具体任务]
【要求】
- [格式要求]
- [风格要求]
- [长度要求]
【背景】[相关背景信息]
【示例】（可选）[参考样例]
```

## 四、不同场景的提示词模板（可直接复制）

```markdown

```

### 场景 2：代码编写

```markdown
- 给出使用示例
【输入】[相关代码或需求描述]
```

### 场景 3：数据分析

```markdown
【任务】分析以下数据并给出洞察
```

## 五、常见提示词错误（我见过的）

### 错误 1：太模糊

```
❌ 帮我写点东西
✅ 帮我写一篇关于 AI 工具的文章，目标读者是职场人士，字数 2000 字
```

### 错误 2：一次问太多

```
❌ 分析数据、写代码、做PPT
✅ 先分析数据，我们一步步来
```

### 错误 3：没有背景信息

```
❌ 怎么解决这个问题？
✅ 我在做[项目]，遇到了[问题]，尝试过[方法]但没用，请问怎么解决？
```

### 错误 4：期望一次完美

```
❌ 写篇文章直接用
✅ 先写初稿，我再根据反馈迭代优化
```

## 六、我的提示词优化技巧

### 技巧 1：迭代对话

**第一次：**
```
写一篇文章大纲
```

**第二次：**
```
根据大纲，先写第一部分
```

**第三次：**
```
继续写第二部分
```

**效果：** 分段控制，质量更高

### 技巧 2：给出示例

```
按照这个格式输出：
【标题】xxx
【摘要】xxx
【正文】xxx
【总结】xxx
```

**效果：** 输出格式更符合预期

### 技巧 3：明确"不要什么"

```
不要：
- 太学术化的表达
- 过度使用比喻
- 空洞的理论
```

**效果：** 避免不需要的内容

## 七、总结

> **提示词的核心：清晰 + 具体 + 有结构**
>
> 你说得越清楚，我输出得越精准。

**提示词公式：**
> 角色 + 任务 + 要求 + 背景 = 高质量输出

**我的建议：**
- 不要追求一次完美，迭代优化
- 给具体的要求，不要模糊
- 有参考样例更好
- 分步骤比一次到位更可控
