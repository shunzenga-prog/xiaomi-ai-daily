## 二、实际操作过程

### 第一步：生成周报草稿

**我的提示词：**
```
你是一个职场周报助手。
用户输入本周工作要点，你输出结构清晰的周报。

要求：
1. 分成"本周完成"、"遇到问题"、"下周计划"三部分
2. 每项工作用一句话概括
3. 语言简洁，适合邮件发送
4. 添加一个本周亮点/成果

输入：
[上面的原始信息]
```

**输出结果：**
```
本周工作周报

【本周完成】
- 完成50份用户调研问卷设计与收集
- 召开3次需求评审会议，推进PRD文档初稿
- 处理2起用户投诉，及时闭环

【遇到问题】
- 需求评审排期紧张，开发资源不足
- 问卷回收率低于预期目标

【下周计划】
- 完善PRD文档细节
- 与开发团队确认最终排期
- 优化问卷回收渠道

【本周亮点】
用户调研问卷从设计到收集仅用时3天，为产品迭代提供了真实数据支撑。
```

### 第二步：优化语言风格

**我的提示词：**
```
上面的周报太正式了，帮我改得更自然一点：
- 去掉括号，用列表
- 语言更口语化但不失专业
- 保持简洁
```

**输出结果：**
```
本周工作周报

本周完成：
1. 用户调研问卷已收集50份，为后续迭代提供数据支持
2. 需求评审会开了3次，PRD初稿已完成
3. 处理2个用户投诉，均已闭环

遇到的问题：
- 开发排期紧张，需要下周再确认
- 问卷回收率偏低，考虑换渠道推

下周计划：
- 完善PRD文档细节
- 和开发确认最终排期
- 追踪问卷回收情况

本周亮点：调研问卷3天完成50份收集，效率不错。
```

### 第三步：对比人工写 vs AI 写

| 维度 | 人工写 | AI 辅助 |
|------|--------|---------|
| 时间 | 约 15-20 分钟 | 约 2 分钟 |
| 结构 | 需要自己组织 | 自动生成框架 |
| 语言 | 反复修改 | 一键优化风格 |
| 亮点提炼 | 可能遗漏 | 自动总结 |

## 三、实际收获

### 3.1 真实效果验证
✅ AI 能在 2 分钟内生成结构清晰的周报
✅ 可以快速切换语言风格（正式/轻松）
✅ 自动提炼"本周亮点"，这是人工容易忽略的

### 3.2 需要注意的问题
⚠️ AI 不知道你的具体工作细节，需要你提供原始信息
⚠️ 生成的周报需要人工检查，避免错误
⚠️ 不同公司周报格式不同，提示词需要调整

### 3.3 给读者的实操建议

**核心提示词模板（复制即用）：**
```
角色：职场周报助手
任务：根据工作要点生成周报
要求：
1. 分三部分：本周完成、遇到问题、下周计划
2. 每项一句话，简洁清晰
3. 自动提炼本周亮点
4. 语言风格可选（正式/轻松）

输入格式：
【本周工作】（你填）
【遇到问题】（你填）
【下周计划】（你填）
```

**使用步骤：**
1. 打开任意 AI 工具（ChatGPT、DeepSeek、豆包等）
2. 复制上面的提示词模板
3. 填入你本周的实际工作要点
4. 生成周报，检查修改后发送

## 四、总结

**实测结论：** 用 AI 写周报确实能提效，从 15 分钟缩短到 2 分钟。
