# 龙虾调度器

智能管理龙虾军团，让每个龙虾都有活干。

## 触发场景

- 用户说"派活"、"分配任务"
- 定时任务检查龙虾状态
- 有新任务需要分配
- 用户问"龙虾在干嘛"

## 龙虾能力矩阵

| 龙虾 | 核心能力 | 适合任务 | 不适合任务 |
|------|----------|----------|------------|
| 🦞 代码龙虾 | 写代码、脚本、工具 | 搭建服务、写脚本、修bug | 纯内容创作 |
| 🔍 研究龙虾 | 搜索、分析、总结 | 热点追踪、竞品分析、学习研究 | 实操执行 |
| 🎨 设计龙虾 | 设计、排版、美化 | 文章排版、封面设计、图片处理 | 代码开发 |
| 📁 档案龙虾 | 整理、归档、清理 | 文件整理、memory维护、清理废弃文件 | 创意工作 |
| 🏄 冲浪龙虾 | 浏览、发现、收集 | 找爆款案例、收集素材、网络冲浪 | 深度分析 |
| 📱 运营龙虾 | 运营、互动、发布 | 检查评论、发布内容、用户互动 | 技术开发 |

## 调度原则

### 1. 优先级匹配

```
P0 任务（紧急） → 立即分配，可用多个龙虾协作
P1 任务（重要） → 分配最适合的龙虾
P2 任务（日常） → 轮流分配，保持活跃
```

### 2. 负载均衡

```
闲置龙虾 → 优先分配
忙碌龙虾 → 等待完成后再分配新任务
超时龙虾 → 记录问题，考虑换龙虾
```

### 3. 协作模式

```
复杂任务 → 多龙虾协作
例：写文章 = 研究龙虾(调研) + 设计龙虾(排版) + 运营龙虾(发布)
```

## 调度流程

```python
# 1. 检查龙虾状态
sessions_list() → 查看活跃龙虾

# 2. 分析任务类型
任务描述 → 匹配龙虾能力

# 3. 分配任务
sessions_spawn(
  label="research-lobster",
  task="追踪今日AI热点",
  model="qwen3.5-plus",
  mode="run"
)

# 4. 记录到 MAINTENANCE.md
更新龙虾状态、任务、开始时间
```

## 任务模板

### 研究龙虾任务模板

```
任务：[研究主题]
输出：reports/[主题]-YYYY-MM-DD.md
要求：
- 搜索至少5个来源
- 提供可操作的结论
- 记录经验教训
```

### 代码龙虾任务模板

```
任务：[开发需求]
输出：scripts/[功能名].js
要求：
- 代码要可运行
- 注释清晰
- 记录使用方法
```

### 冲浪龙虾任务模板

```
任务：[搜索目标]
输出：reports/[收集结果].md
要求：
- 找至少10个案例
- 提供链接和截图
- 总结特点
```
