# 公众号爆款文章创作

专为公众号平台优化的深度文章创作技能。

## 触发场景

- "写一篇公众号文章"
- "发公众号"
- "公众号创作"

## ⚠️ 核心原则（必读）

### 爆款公式
```
爆款 = 热点选题 × 情绪价值 × 结构化表达 × 视觉呈现
```

### 三大禁忌
1. ❌ **表格太多** - 像产品说明书，不像在聊天
2. ❌ **论述太浅** - 只罗列信息，没有深入分析
3. ❌ **缺少真实感** - 没有实践、没有情感、没有金句

### 小咪特色（必须有）
- 🐱 小咪身份标识
- ✅ 真实实践过程（"我试过了..."）
- 💡 可转发金句（每篇至少 1 句）
- 🎯 明确行动号召（结尾必须有）

---

## 创作流程

1. **确认选题** → 热点/教程/观点
2. **验证时效** → ⚠️ 新闻类必查发布时间
3. **写标题** → 爆款公式（见下）
4. **写开头** → 黄金 3 秒钩子
5. **写正文** → SCQA 结构，口语化
6. **写结尾** → 行动号召
7. **自检** → 发布前检查清单

---

## ⚠️ 时效验证（搜资料必做）

| 发布时间 | 使用建议 |
|----------|----------|
| < 24h | ✅ 可放心使用 |
| 24-48h | ⚠️ 需补充最新进展 |
| > 48h | 💡 转分析角度，不当新闻 |
| 已辟谣 | ❌ 不使用 |

## 爆款标题（11 个套路）

| 套路 | 公式 | 案例 |
|------|------|------|
| 数字型 | 数字 + 核心信息 | "3 小时搞定一周工作量" |
| 对比型 | A vs B 强烈反差 | "卡里只有 10 万，下单 600 万房子" |
| 疑问型 | 如何/为什么 + 痛点 | "如何用 AI 写出 10 万 + 爆文？" |
| 场景型 | 具体场景 + 结果 | "凌晨 3 点，我靠 AI 赚了第一桶金" |
| 教程型 | 手把手/保姆级 | "保姆级教程：5 分钟上手 DeepSeek" |
| 清单型 | N 个方法/技巧 | "10 个让效率翻倍的 AI 工具" |
| 揭秘型 | 内幕/秘密/真相 | "AI 行业不愿告诉你的 5 个真相" |
| 预测型 | 202X 年趋势 | "2026 年，这 3 个职业最吃香" |
| 人群型 | 特定人群 + 利益 | "程序员必看：AI 时代生存指南" |
| 结果型 | 时间 + 惊人结果 | "7 天，我用 AI 写出百万阅读" |
| 复合型 | 2-3 个套路组合 | "全网粉丝 1800 万，这位博主如何玩转流量？" |

**字数控制**：20-30 字最佳

## 🎯 黄金开头（3 秒留人）

**核心目标**：3 秒内留住读者，65% 用户在 3 秒内决定是否继续阅读

### 9 个开头钩子

| 类型 | 公式 | 案例 |
|------|------|------|
| 提问法 | "你知道...吗？" | "你知道 90% 的 AI 工具都在浪费时间吗？" |
| 数据法 | 惊人数据 + 解读 | "1000 篇爆文分析，发现一个共同秘密" |
| 故事法 | 场景 + 人物 + 冲突 | "上周三，我收到一条改变职业生涯的消息" |
| 对比法 | Before/After | "以前写文章要 3 小时，现在只要 15 分钟" |
| 引用法 | 名人名言 + 解读 | "马斯克说：AI 是人类最大的威胁。但..." |
| 痛点法 | 直击读者痛点 | "你是不是也遇到过这种情况：..." |
| 粉丝法 | "最近很多粉丝问..." | "最近很多粉丝问我：AI 写作到底靠不靠谱？" |
| 悬念法 | 制造悬念 | "这篇文章，我犹豫了很久要不要发" |
| 结论法 | 先给结论 | "直接说结论：2026 年，不会用 AI 的人将被淘汰" |

### 开头段落结构
```
第 1 句：钩子（吸引注意）
第 2-3 句：背景/问题阐述
第 4 句：本文价值预告
第 5 句：引导继续阅读
```

## 正文结构（SCQA 模板）

### SCQA 结构（麦肯锡金字塔原理）
```
S（Situation）情境：大家都熟悉的事实引入
C（Complication）冲突：实际情况的矛盾/问题
Q（Question）疑问：那该怎么办？
A（Answer）答案：给出解决方案
```

### 段落规范

| 要求 | 标准 |
|------|------|
| 段落长度 | 每段 3-5 行，不超过手机一屏 |
| 小标题 | 每 300-500 字一个，用加粗问句 |
| 信息密度 | 每屏至少 1 个核心观点 |
| 金句 | 每 800 字至少 1 个可转发金句 |

### ⚠️ 去掉 AI 味道

| ❌ 教科书式 | ✅ 博主式 |
|------------|----------|
| 一、发生了什么 | 先看发生了什么。 |
| 二、为什么重要 | **为什么这很重要？** |
| 三、怎么做 | **该怎么操作？** |
| 四、小咪的观点 | 写到这，小咪想分享几个核心观察。 |

### ⚠️ 减少表格使用

**原则**：表格 ≤ 2 个/篇，优先用文字叙述

| ✅ 可以用表格 | ❌ 别用表格 |
|--------------|------------|
| 数据对比（价格、性能） | 描述性内容 |
| 功能清单 | 故事、案例 |
| 步骤流程 | 观点论述 |

**替代方案**：用加粗 + 自然叙述

## 结尾（行动号召）

### 6 个结尾模板

| 类型 | 公式 | 案例 |
|------|------|------|
| 总结式 | 核心要点回顾 | "以上就是...的 3 个关键方法" |
| 提问式 | 引发思考 | "你觉得 AI 会取代人类吗？" |
| 行动式 | 明确行动号召 | "现在就去试试，评论区告诉我结果" |
| 金句式 | 一句点睛 | "未来淘汰你的不是 AI，是会使用 AI 的人" |
| 预告式 | 下期内容 | "下期揭秘：如何用 AI 打造自动化工作流" |
| 互动式 | 引导留言 | "你在用哪些 AI 工具？欢迎留言分享" |

### 结尾必备要素
1. **核心观点重申**（1 句话）
2. **金句/启发**（可选）
3. **行动号召**（必须有）

## 📐 排版规范

### 核心参数

| 元素 | 推荐值 |
|------|--------|
| 正文字号 | 15-16px |
| 行距 | 1.75-2.0 |
| 段落长度 | ≤5 行 |
| 表格数量 | ≤2 个/篇 |

### 标题样式

**H2 标题**（章节主标题）：
```html
<h2 style="font-size:18px;font-weight:bold;color:#1A1A1A;border-left:4px solid #1E88E5;padding-left:12px;margin:35px 0 15px 0;">标题文字</h2>
```

**H3 标题**（小节标题）：
```html
<h3 style="font-size:16px;font-weight:bold;color:#333;margin:25px 0 12px 0;">标题文字</h3>
```

### 重点标注

**加粗强调**：
```html
<span style="font-weight:bold;color:#E74C3C;">重点内容</span>
```

**底色高亮**：
```html
<span style="background-color:#FEF3C7;padding:2px 4px;border-radius:2px;">重点内容</span>
```

### 引用框

```html
<div style="border-left:4px solid #1E88E5;padding:15px 20px;margin:20px 0;background:#F8F9FA;color:#555;font-style:italic;">
  引用内容...
</div>
```

### 配图规范

| 位置 | 尺寸 |
|------|------|
| 封面图 | 900×383px |
| 文中图 | 1080px 宽 |

**图片代码**：
```html
<img src="URL" style="width:100%;max-width:1080px;height:auto;display:block;margin:20px auto;">
<div style="font-size:13px;color:#999;text-align:center;margin-top:10px;">图：说明文字</div>
```

### 颜色搭配

| 用途 | 颜色 | 代码 |
|------|------|------|
| 正文 | 深灰 | #1A1A1A |
| 标题/链接 | 科技蓝 | #1E88E5 |
| 重点/警示 | 红色 | #E74C3C |
| 成功/正面 | 绿色 | #27AE60 |

**原则**：全文颜色 ≤4 种

### 内容检查
- [ ] 标题：20-30 字，有吸引力？
