# 论述技巧速查手册

> 基于「论述技巧研究报告」，用于创作时快速自检

---

## 零、写作风格要求（⚠️ 最重要）

**去掉 AI 味道，用真人博主的表达方式！**

### 禁止的教科书式表达

| ❌ AI 味重 | ✅ 真人感 |
|-----------|----------|
| 一、发生了什么 | 先看发生了什么。 |
| 二、为什么重要？ | **为什么这很重要？** |
| 三、怎么做？ | **该怎么操作？** |
| 四、我的观点 | 写到这，想分享几个核心观察。 |
| ### 1️⃣ 第一点 | **第一，...** |
| ### 2️⃣ 第二点 | **第二，...** |

### 核心原则

1. **用加粗问句过渡**，不用编号标题
2. **用自然句子衔接**，像跟读者聊天
3. **避免列表式表达**，改成流动叙述
4. **观点引入用"写到这..."**，不用独立章节

### 示例对比

**教科书式（AI味）**：
```
## 一、背景介绍
发生了 X 事件...

## 二、深度分析
这很重要因为...

## 三、实践建议
第一，...
第二，...
第三，...
```

**博主式（真人感）**：
```
先看发生了什么。

发生了 X 事件...

**为什么这很重要？**

这很重要因为...

**该怎么操作？**

这里给几条建议：

**第一，...**

**第二，...**

写到这，想分享几个核心观察。
```

## 一、三层论述法（核心方法）

每个重要观点必须包含三层：

```
【观点】清晰陈述观点
【支撑】数据/案例/引用作为支撑
【解释】支撑材料如何证明观点 + 对读者的意义
```

### 示例

❌ 论述不足：
> 大模型正在快速发展

✅ 论述充分：
> 【观点】大模型正在快速发展  
> 【支撑】2025 年 Q1 全球大模型参数量平均增长 3.2 倍，迭代周期从 6 个月缩短至 2 个月（来源：AI Index Report 2025）  
> 【解释】这意味着技术迭代进入"月级"时代，跟不上节奏的厂商将在 3-6 个月内失去竞争力

## 二、5W1H 检查清单

| 维度 | 检查问题 | 达标标准 |
|-----|---------|---------|
| What | 核心事件说清楚了吗？ | 非专业人士能看懂 |
| Why | 为什么重要/会发生？ | 至少 2 个原因分析 |
| Who | 涉及哪些关键角色？ | 主要参与者提及 |
| When | 时间线清晰吗？ | 关键时间点明确 |
| Where | 应用场景/影响范围？ | 至少 1 个具体场景 |
| How | 怎么实现/怎么做？ | 有步骤或机制说明 |

**总分 ≥ 10/12 = 可发布**

---

## 三、对比强化模板

关键观点处强制加入对比：

```markdown
**对比视角**：
| 维度 | 之前 | 现在 | 变化 |
|------|------|------|------|
| [指标1] | [旧数据] | [新数据] | [变化率] |
| [指标2] | [旧做法] | [新做法] | [核心差异] |

**这意味着**：[对读者的实际影响]
```

### 应用场景

- 技术升级 → 性能/成本对比
- 政策变化 → 新旧政策对比
- 市场趋势 → 历年数据对比
- 产品迭代 → 版本功能对比

## 四、资料验证规范（⚠️ 必须验证）

**搜索参考资料时，必须验证来源和时间，避免使用过期消息！**

### 验证流程（搜资料时）

```
1. 搜索原文发布时间
2. 判断是否过时（> 48h 要谨慎）
3. 检查是否有最新进展/辟谣/反转
4. 决定是否使用该资料
```

### 时效判断标准（筛选资料用）

| 发布时间 | 分类 | 使用建议 |
|----------|------|----------|
| **< 24h** | 🔥 实时热点 | 可放心使用 |
| **24-48h** | 📈 近期热点 | 需补充最新进展 |
| **> 48h** | 💡 长青话题 | 转分析角度，不当新闻写 |
| **已辟谣/反转** | ❌ 过时/虚假 | 不使用 |

### ⚠️ 注意

**创作内容不需要强制标注来源时间**，这不是常规做法。

验证是用于**筛选参考资料**，确保不使用过期消息。

## 五、深度追问法

对每个重要陈述追问 3 层：

```
陈述：AI 正在改变内容创作

追问 1：怎么改变的？
→ 生成效率提升、成本降低

追问 2：提升多少？
→ 效率提升 5-10 倍，成本降低 70%

追问 3：有什么证据？
→ 某公司案例 + 行业报告数据

追问不到第 3 层 = 论述深度不够
```

## 六、发布前自检清单

文章发布前 5 分钟检查：

- [ ] 每个核心观点有数据/案例支撑？
- [ ] 有至少 1 个对比分析？
- [ ] 重要数据来源标注完整？（机构 + 时间 + 链接）
- [ ] 有"这意味着什么"的深度分析？
- [ ] 有"具体怎么做"的可操作建议？

**3 个以上 ❌ = 需补充论述后再发布**
