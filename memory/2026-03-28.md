### 22:25 - 深入排查 web_fetch 问题

**根本原因找到：**
- WSL DNS (10.255.255.254) 把 github.com 解析到私有 IP (198.18.0.248)
- OpenClaw web_fetch 有安全检查，阻止私有 IP 访问
- Google DNS 显示真实 IP 是 20.27.177.113

**问题链：**
```
WSL 内网 DNS → 解析到私有 IP → OpenClaw 安全阻止 → web_fetch 不可用
```

**解决方案（需 sudo）：**
1. 修改 /etc/resolv.conf 为公共 DNS (8.8.8.8, 1.1.1.1)
2. 修改 /etc/wsl.conf 防止自动覆盖
3. 或添加 hosts 映射

**生成报告：** reports/web-fetch-blocked-solution-2026-03-28.md

**boss 提醒：** 遇到问题要优先解决，解决不了再尝试别的方法

**已修复：**
1. ✅ 修改 /etc/wsl.conf（禁用自动生成 resolv.conf）
2. ✅ 修改 /etc/resolv.conf（使用公共 DNS）
3. ✅ 添加 hosts 映射（GitHub 相关域名）

**修复结果：**
