## 二、内容策略分析

### 2.1 内容类型分布

```
行业媒体（机器之心/量子位）：
├─ 快讯资讯：40%（短平快，蹭热点）
├─ 深度解读：30%（技术/产业分析）
├─ 论文解读：15%（学术研究）
├─ 人物访谈：10%（大佬专访）
└─ 资源整理：5%（工具/课程）

个人 IP（小咪的 AI 日常 - 目标定位）：
├─ 场景教程：40%（职场/生活应用）
├─ 工具测评：25%（对比/推荐）
├─ 实战案例：20%（真实使用经历）
├─ 观点输出：10%（趋势/思考）
└─ 资源分享：5%（模板/资料）
```

### 2.2 选题方向对比

| 竞品选题 | 特点 | 小咪差异化策略 |
|----------|------|----------------|
| 大模型技术突破 | 专业性强，门槛高 | 聚焦「普通人如何用」 |
| 融资/产业新闻 | 离用户远，看个热闹 | 聚焦「对我有什么用」 |

| 大佬访谈 | 仰望视角，难复制 | 聚焦「普通人可复制的路径」 |

**小咪标题策略（差异化）：**
- 身份共鸣：《打工人》《小白》《普通人》
- 场景具体：《写周报》《做 PPT》《回邮件》

## 八、创作策略 V4（更新版）

### 8.1 标题策略

**核心公式：**
```
[身份标签] + [具体场景] + [量化结果] + [情绪词]

示例：
《打工人必备！5 个 AI 自动化工作流，每天省下 2 小时》
《小白也能用！这个 AI 工具让我效率提升 3 倍》
《别再加班了！用 AI 写周报，准点下班不是梦》
```

**标题检查清单：**
- [ ] 是否包含具体数字？
- [ ] 是否有身份标签（打工人/小白/普通人）？
- [ ] 是否有量化结果（省 X 小时/提升 X 倍）？
- [ ] 是否有情绪词（必备/别再/终于）？
- [ ] 是否控制在 25 字以内？

### 8.2 开头策略

**3 秒钩子公式：**
```
【场景痛点】+【共鸣描述】+【解决方案预告】

示例：
「你有没有经历过这样的时刻：
凌晨 3 点，还在为周报发愁；
明明工作完成了，却要花 2 小时写汇报；
第二天还要早起开会，整个人都快崩溃了。

别担心，今天小咪分享一个方法，
用 AI 写周报，10 分钟搞定，准点下班不是梦。」
```

### 8.3 结构策略

**标准结构（5 段式）：**
```
1. 痛点场景（300 字）- 让读者说「这就是我」
2. 解决方案（200 字）- 核心方法预告
3. 详细步骤（800 字）- 可复制的操作流程
4. 案例佐证（400 字）- 真实使用效果
5. 行动号召（200 字）- 下一步做什么 + 互动
```

### 8.4 金句策略

**每篇文章至少 3 句金句：**
- 开头定调句（建立期待）
- 中部记忆点（强化认知）
- 结尾转发句（适合引用）

**金句公式：**
```
「不是...而是...」
「当...时，...」
「真正的...从来不是...而是...」
```

### 8.5 互动策略

**结尾固定互动：**
```
【提问式】你在 XX 场景用过 AI 吗？效果怎么样？
【投票式】A vs B，你更喜欢哪个？
【分享式】你有什么 AI 使用技巧？评论区分享~
【资料式】关注 + 回复「XX」领取资源包
```

## 十、总结

### 核心发现

1. **行业媒体**强在资讯速度和权威性，但离普通用户远
2. **个人 IP**强在专业深度，但门槛高、不够亲切
3. **小咪机会**在生活化场景 + 亲切人设 + 持续成长叙事

### 差异化策略

```
不做「最专业」的 AI 号，
做「最懂你」的 AI 伙伴。

不追「最热」的热点，
解决「最痛」的场景。

不写「最难」的教程，
分享「最有用」的技巧。
```

### 一句话总结

**用 AI 的视角写 AI，用伙伴的语气帮伙伴。**
