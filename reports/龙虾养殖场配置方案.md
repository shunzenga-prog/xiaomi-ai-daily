# 🦞 小咪的龙虾养殖场 - 多 Agent 配置方案

## 🎯 目标

配置多个 OpenClaw Agent，形成"龙虾养殖场"，实现分工协作。

---

## 📋 龙虾阵容

| 龙虾名 | 职责 | 模型 | 优先级 |
|--------|------|------|--------|
| **小咪 (CEO)** | 统筹 + 内容 + 运营 | qwen3.5-plus | P0 |
| **龙虾 - 代码** | 写代码 + 调试 | qwen3-coder-next | P1 |
| **龙虾 - 研究** | 深度调研 | qwen3-max | P2 |
| **龙虾 - 设计** | 配图 + 排版 | qwen3.5-plus | P3 |

---

## 🔧 配置方案

### 方案 A：单 Gateway 多 Agent（推荐）

**优点：**
- ✅ 共享配置和工具
- ✅ 统一管理
- ✅ 资源利用高效

**配置位置：** `~/.openclaw/openclaw.json`

```json
{
  "agents": {
    "list": [
      {
        "id": "xiaomi",
        "name": "小咪",
        "model": "qwen3.5-plus",
        "role": "内容创作 + 运营 + 统筹",
        "workspace": "/home/zengshun/.openclaw/workspace"
      },
      {
        "id": "code-lobster",
        "name": "代码龙虾",
        "model": "qwen3-coder-next",
        "role": "代码开发 + 调试 + 审查",
        "workspace": "/home/zengshun/.openclaw/workspace/code"
      },
      {
        "id": "research-lobster",
        "name": "研究龙虾",
        "model": "qwen3-max-2026-01-23",
        "role": "深度调研 + 分析报告",
        "workspace": "/home/zengshun/.openclaw/workspace/research"
      }
    ]
  }
}
```

### 方案 B：多 Gateway 实例

**优点：**
- ✅ 完全隔离
- ✅ 独立配置
- ✅ 可以部署在不同机器

**缺点：**
- ❌ 资源占用多
- ❌ 配置复杂

---

## 🚀 实施步骤

### 第 1 步：备份配置
```bash
cp ~/.openclaw/openclaw.json ~/.openclaw/openclaw.json.bak
```

### 第 2 步：修改配置
编辑 `~/.openclaw/openclaw.json`，添加多 Agent 配置

### 第 3 步：重启 Gateway
```bash
openclaw gateway restart
```

### 第 4 步：测试协作
```bash
# 测试小咪
openclaw agent xiaomi "你好，我是小咪"

# 测试代码龙虾
openclaw agent code-lobster "帮我写一个 Python 脚本"

# 测试研究龙虾
openclaw agent research-lobster "调研一下 AI Agent 趋势"
```

## 📊 协作模式

### 模式 1：任务分发
```
小咪接收任务 → 分析 → 分发给合适的龙虾 → 汇总结果
```

### 模式 2：并行处理
```
大任务 → 拆分成子任务 → 多龙虾并行 → 合并结果
```

### 模式 3：流水线
```
研究龙虾调研 → 小咪写文章 → 代码龙虾写示例 → 设计龙虾配图
```
