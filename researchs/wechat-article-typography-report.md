## 一、核心排版参数推荐

### 1.1 字号设置

| 内容类型 | 推荐字号 | 说明 |
|---------|---------|------|
| 正文 | 15px - 16px | 主流选择，阅读舒适 |
| 标题 H1 | 18px - 20px | 主标题，加粗 |
| 标题 H2 | 17px - 18px | 次级标题，加粗 |
| 标题 H3 | 16px - 17px | 三级标题，可加粗 |
| 注释/说明 | 13px - 14px | 图片说明、引用来源 |
| 重点强调 | 15px - 16px | 加粗或变色，不改变字号 |

**趋势观察**: 2025-2026 年主流公众号趋向使用 **15px** 作为正文字号，兼顾清晰度与信息密度。

### 1.2 行距设置

| 场景 | 推荐行距 | 倍数 |
|-----|---------|------|
| 正文 | 1.75 - 2.0 | 最舒适阅读体验 |
| 密集信息 | 1.6 - 1.75 | 技术文档、代码说明 |
| 宽松排版 | 2.0 - 2.2 | 文艺类、情感类内容 |

**最佳实践**: **1.75 倍行距** 是科技类公众号的黄金标准。

### 1.3 字间距

| 设置 | 推荐值 |
|-----|--------|
| 正文字间距 | 0 - 1px |
| 标题字间距 | 1 - 2px |
| 英文/数字间距 | 0.5 - 1px |

**注意**: 微信公众号编辑器默认字间距为 0，一般不需要额外调整。

### 1.4 段间距

| 类型 | 推荐值 |
|-----|--------|
| 段落间距 | 15px - 20px |
| 空行 | 使用 `<br>` 或空段落 |
| 模块间距 | 30px - 40px |

```
H1: 文章主标题（通常由公众号封面和标题决定）

H3: 小节标题 ██████
```

### 2.2 顶级公众号标题样式分析

#### 量子位 (QbitAI)

  ```

  ```
- **颜色**: 深蓝色 (#1E3A8A) 或品牌色
- **字号**: 18px，加粗

  ```

  ```
- **颜色**: 品牌红色 (#C41E3A) 背景，白色文字
- **字号**: 17px，加粗
- **内边距**: 8px 12px

#### 差评 (chaping38)
- **H2 样式**: 简约文字 + 下划线
  ```
  标题文字
  ────────
  ```
- **颜色**: 黑色 (#000000)
- **字号**: 18px，加粗
- **下边距**: 20px

## 三、重点标注方式

### 3.1 加粗强调

**使用场景**: 核心观点、关键数据、结论句

**规范**:
- 单段加粗内容不超过 3 处
- 避免整段加粗
- 加粗文字前后留白

**示例**:
> 这项技术的**准确率达到了 98.5%**，远超行业平均水平。

### 3.2 颜色标注

**推荐配色方案**:

| 用途 | 颜色代码 | 说明 |
|-----|---------|------|
| 主强调色 | #E74C3C | 红色，警示/重要 |
| 次强调色 | #3498DB | 蓝色，链接/参考 |
| 成功/正面 | #27AE60 | 绿色，积极信息 |
| 警告/注意 | #F39C12 | 橙色，提醒 |

**使用原则**:
- 全文强调色不超过 2 种
- 避免使用亮黄、荧光色
- 确保色盲友好（红绿不同时使用）

### 3.3 底色高亮

**使用场景**: 金句、核心结论、引用

**推荐样式**:
```css
.highlight {
  background-color: #FEF3C7; /* 浅黄色 */
  padding: 2px 4px;
  border-radius: 2px;
}

.highlight-blue {
  background-color: #DBEAFE; /* 浅蓝色 */
  padding: 2px 4px;
  border-radius: 2px;
}
```

## 四、引用框与分割线

### 4.1 引用框样式

**样式 A - 左侧线型** (最常用)
```html
<div style="
  border-left: 4px solid #1E88E5;
  padding: 15px 20px;
  margin: 20px 0;
  background: #F8F9FA;
  color: #555;
  font-style: italic;
">
  引用内容...
</div>
```

**样式 B - 完整边框型**
```html
<div style="
  border: 1px solid #E0E0E0;
  border-radius: 8px;
  padding: 20px;
  margin: 20px 0;
  background: #FAFAFA;
">
  引用内容...
</div>
```

**样式 C - 引用来源型**
```html
<div style="
  border-left: 3px solid #9E9E9E;
  padding: 15px 20px;
  margin: 20px 0;
  color: #666;
">
  引用内容...
  <div style="
    margin-top: 10px;
    font-size: 13px;
    color: #999;
    text-align: right;
  ">—— 来源说明</div>
</div>
```

### 4.2 分割线使用

**规范**:
- 章节之间使用，同章节内避免
- 单篇文章不超过 5 条
- 风格统一（颜色、粗细）

**推荐样式**:
```html
<!-- 简约型 -->
<hr style="
  border: none;
  border-top: 1px solid #E0E0E0;
  margin: 40px 0;
">

<!-- 装饰型 -->
<hr style="
  border: none;
  border-top: 2px dashed #BDBDBD;
  margin: 40px 0;
">

<!-- 品牌色型 -->
<hr style="
  border: none;
  border-top: 3px solid #1E88E5;
  width: 100px;
  margin: 40px auto;
">
```

## 五、配图规范

### 5.1 图片尺寸

| 位置 | 推荐尺寸 | 说明 |
|-----|---------|------|
| 封面图 | 900×383px | 官方推荐比例 2.35:1 |
| 文首图 | 1080px 宽 | 高度自适应，建议 600-800px |
| 文中图 | 1080px 宽 | 高度自适应 |
| 信息图 | 1080px 宽 | 确保文字清晰可读 |
| 缩略图 | 200×200px | 多图集场景 |

### 5.2 图片位置

**最佳实践**:
- 文首：1 张吸引眼球的主图
- 章节开头：每 H2 标题后可配 1 图
- 长段落：每 300-500 字可插入 1 图
- 文末：总结图或二维码引导

**密度建议**:
- 短文 (1500 字内): 2-4 张图
- 中篇 (1500-3000 字): 4-8 张图
- 长文 (3000 字+): 8-15 张图

### 5.3 图片说明

**规范**:
- 字号：13px
- 颜色：#999999
- 位置：图片正下方，居中
- 格式：`图：说明文字` 或 `来源：XXX`

```html
<div style="
  font-size: 13px;
  color: #999;
  text-align: center;
  margin-top: 10px;
">图：说明文字</div>
```

### 5.4 图片风格统一

**顶级公众号做法**:
- **量子位**: 科技感蓝色调，统一滤镜
- **机器之心**: 红色品牌色边框或角标
- **差评**: 年轻化插画风格，高饱和度

**建议**:
1. 建立品牌色卡，图片调色统一
2. 使用统一的水印/角标位置
3. 保持图片风格与内容调性一致

## 六、颜色搭配方案

### 6.1 主色选择

**科技类公众号推荐**:

| 风格 | 主色 | 辅助色 | 适用场景 |
|-----|------|--------|---------|
| 科技蓝 | #1E88E5 | #1565C0, #BBDEFB | AI、科技、互联网 |
| 深邃黑 | #1A1A1A | #424242, #E0E0E0 | 严肃、专业内容 |
| 活力红 | #E53935 | #C62828, #FFCDD2 | 热点、资讯类 |
| 清新绿 | #43A047 | #2E7D32, #C8E6C9 | 环保、健康类 |

### 6.2 全文配色规范

**60-30-10 原则**:
- **60% 主色**: 正文、背景（黑/深灰）
- **30% 辅助色**: 标题、引用框（品牌色）
- **10% 强调色**: 重点标注、按钮（亮色）

**推荐配色组合**:

```
方案 A - 专业科技风
主色：#1A1A1A (正文)
辅助色：#1E88E5 (标题、链接)
强调色：#FF6B6B (重点)
背景：#FFFFFF

方案 B - 温暖阅读风
主色：#2C2C2C (正文)
辅助色：#D4A574 (标题)
强调色：#E76F51 (重点)
背景：#FEFAF0

方案 C - 极简黑白风
主色：#000000 (正文)
辅助色：#666666 (次级)
强调色：#000000 (加粗)
背景：#FFFFFF
```

### 6.3 避免的配色

❌ 高饱和度背景色（阅读疲劳）
❌ 红绿同时大量使用（色盲不友好）
❌ 超过 4 种主色调（视觉混乱）
❌ 浅灰文字配白色背景（对比度不足）

## 七、移动端阅读体验优化

### 7.1 段落控制

**规范**:
- 单段不超过 5 行（手机屏幕）
- 单行不超过 25 字
- 长段落主动拆分

**对比**:
```
❌ 差：一段 10 行，密密麻麻
✅ 好：2-3 行一段，呼吸感强
```

### 7.2 留白艺术

**关键留白位置**:
- 文首标题上：30px
- 章节之间：40px
- 图片上下：20px
- 引用框内外：15px
- 文末：50px（引导关注）

### 7.3 适配不同屏幕

**测试方法**:
1. 使用微信开发者工具预览
2. 实际手机发送预览测试
3. 检查 iOS 和 Android 显示差异

**常见问题**:
- 图片过宽溢出 → 设置 max-width: 100%
- 表格横向滚动 → 使用响应式表格
- 代码块换行 → 设置 overflow-x: auto

### 7.4 暗黑模式适配

**2025-2026 趋势**: 越来越多用户开启深色模式

**适配建议**:
```css
/* 避免使用纯黑背景 */
background: #1A1A1A; /* 而非 #000000 */

/* 文字使用相对亮度 */
color: #E0E0E0; /* 而非 #FFFFFF */

/* 图片添加深色模式滤镜 */
img { filter: brightness(0.9); }
```
